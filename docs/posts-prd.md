# Product Requirements Document: Posts Feed

**Version**: 1.0
**Date**: 2026-05-04
**Author**: Nikita (Product Owner)

---

## Executive Summary

Посты — это лёгкий формат публикаций в Roasti, дополняющий лонгриды (статьи). В отличие от статей со свободной структурой блоков, пост — это короткий контент: фотографии, текст до 500 слов и необязательная ссылка на рецепт. У постов есть рейтинговая система в стиле YouTube: лайк (+1) или дизлайк (−1), рейтинг отражает общую оценку сообщества.

Фича закрывает потребность в быстрой публикации без сложного редактора — поделиться фото чашки, коротким наблюдением или ссылкой на рецепт.

---

## Problem Statement

**Текущая ситуация:** Статьи требуют структуры (заголовок, блоки) и подходят для развёрнутого контента. Для коротких постов (одна фотография, пара предложений) этот формат избыточен.

**Предлагаемое решение:** Отдельный формат постов с упрощённой структурой и механикой лайк/дизлайк.

**Бизнес-эффект:** Снижение порога входа для авторов; рост количества публикаций и вовлечённости.

---

## Success Metrics

Конкретные числовые цели не определены на этом этапе. Рекомендуется отслеживать:

- Количество созданных постов за период (активность авторов)
- Распределение рейтингов (доля постов с положительным рейтингом)
- Количество комментариев на пост (глубина вовлечённости)
- Доля активных пользователей, поставивших лайк хотя бы раз (охват механики)

**Валидация:** Провести ревью метрик через 4 недели после запуска.

## User Personas

### Обычный пользователь
- **Цели:** Быстро поделиться фото из кофейни, коротким впечатлением или вопросом
- **Поведение:** Создаёт короткие посты, лайкает, комментирует

### Обжарщик / Инфлюенсер
- **Цели:** Анонсировать новый лот, поделиться фото производства
- **Поведение:** Публикует посты с фотографиями и ссылкой на рецепт

## User Stories & Acceptance Criteria

### Story 1: Создание поста

**As a** авторизованный пользователь  
**I want to** создать пост с фото и/или текстом  
**So that** быстро поделиться контентом с сообществом

**Acceptance Criteria:**
- [ ] Пост должен содержать хотя бы одно из: фотографию или текст
- [ ] Текст ограничен 500 словами; превышение возвращает ошибку валидации
- [ ] Фотографий может быть одна или несколько
- [ ] Ссылка на рецепт необязательна; можно прикрепить не более одного рецепта
- [ ] Пост сразу публичный — черновиков нет (v1)
- [ ] Любой авторизованный пользователь может создать пост

### Story 2: Просмотр постов

**As a** пользователь  
**I want to** просматривать посты  
**So that** видеть что публикуют другие

**Acceptance Criteria:**
- [ ] Посты всех пользователей доступны, отсортированы по дате (новые первые)
- [ ] Список пагинирован
- [ ] Каждый пост содержит: фотографии, текст, ссылку на рецепт, автора, дату, рейтинг, реакцию текущего пользователя, количество комментариев

### Story 3: Редактирование и удаление поста

**As a** автор поста  
**I want to** отредактировать или удалить свой пост  
**So that** поддерживать актуальность контента

**Acceptance Criteria:**
- [ ] Только автор может редактировать и удалять пост
- [ ] Редактирование доступно без ограничений по времени
- [ ] Редактирование возвращает обновлённый объект поста

### Story 4: Лайк и дизлайк поста

**As a** пользователь  
**I want to** поставить лайк или дизлайк посту  
**So that** выразить отношение к контенту

**Acceptance Criteria:**
- [ ] `POST /posts/{id}/like` — ставит лайк; повторный запрос убирает лайк (toggle)
- [ ] `POST /posts/{id}/dislike` — ставит дизлайк; повторный запрос убирает дизлайк (toggle)
- [ ] Лайк и дизлайк взаимоисключающие: постановка одного убирает другой
- [ ] Ответ содержит актуальный `rating` и `user_reaction` (`"like"` | `"dislike"` | `null`)
- [ ] Лайкнуть/дизлайкнуть может любой авторизованный пользователь, включая автора

### Story 5: Лента лайкнутых постов

**As a** пользователь  
**I want to** просмотреть посты, которые я лайкнул  
**So that** вернуться к понравившемуся контенту

**Acceptance Criteria:**
- [ ] `GET /api/v1/users/{user_id}/likes?type=post` возвращает пагинированный список `LikedPost`
- [ ] Каждый элемент содержит `liked_at` и полный объект поста

### Story 6: Комментарии к посту

**As a** пользователь  
**I want to** оставить комментарий к посту  
**So that** участвовать в обсуждении

**Acceptance Criteria:**
- [ ] Комментарии видны когда пользователь открывает экран поста
- [ ] Правила и поведение комментариев описаны в `comments-prd.md`

## Functional Requirements

### Структура поста

| Поле | Тип | Обязательность | Описание |
|------|-----|----------------|----------|
| `id` | string | — | Идентификатор поста |
| `author` | User | — | Автор |
| `text` | string | Нет | Текст, максимум 500 слов |
| `photos` | string[] | Нет | Массив ID загруженных изображений |
| `recipe_id` | string | Нет | Ссылка на рецепт в Roasti |
| `rating` | int | — | Лайки минус дизлайки; может быть отрицательным |
| `user_reaction` | string\|null | — | Реакция текущего пользователя: `"like"`, `"dislike"` или `null` |
| `comments_count` | int | — | Количество комментариев |
| `created_at` | datetime | — | Дата публикации |
| `updated_at` | datetime | — | Дата последнего редактирования |

**Ограничения:** пост валиден, если содержит хотя бы `text` (непустой) или хотя бы одну фотографию в `photos`.

### Рейтинговая система

- Начальный рейтинг поста — 0
- Лайк добавляет +1, дизлайк добавляет −1 к рейтингу
- Каждый пользователь имеет не более одной активной реакции на пост
- Повторный запрос того же эндпоинта снимает реакцию (toggle)
- Запрос противоположного эндпоинта заменяет реакцию
- Рейтинг — целое число, может быть отрицательным
- Дизлайк анонимный: автор не получает уведомление о дизлайке

### Уведомления

- Лайк поста генерирует уведомление автору (если лайкает не сам автор)
- Дизлайк уведомление не генерирует

### Обработка недоступного рецепта

Бэкенд резолвит `recipe_id` при отдаче поста и добавляет поле `recipe_status: "available" | "unavailable"`. Клиент показывает плейсхолдер если статус `unavailable`. Причина недоступности не раскрывается.

### Видимость постов

Все посты публичные. Авторизация не влияет на видимость.

### Права доступа

| Действие | Кто может |
|----------|-----------|
| Создать пост | Любой авторизованный пользователь |
| Редактировать пост | Только автор |
| Удалить пост | Только автор |
| Лайкнуть / дизлайкнуть | Любой авторизованный пользователь |
| Комментировать | Любой авторизованный пользователь |

## API Contract

| Метод | Путь | Статус | Описание |
|-------|------|--------|----------|
| `GET` | `/api/v1/posts` | 200 | Лента постов (пагинация) |
| `POST` | `/api/v1/posts` | 201 | Создать пост |
| `GET` | `/api/v1/posts/{post_id}` | 200 | Получить пост |
| `PUT` | `/api/v1/posts/{post_id}` | 200 | Обновить пост |
| `DELETE` | `/api/v1/posts/{post_id}` | 204 | Удалить пост |
| `POST` | `/api/v1/posts/{post_id}/like` | 200 | Toggle лайк |
| `POST` | `/api/v1/posts/{post_id}/dislike` | 200 | Toggle дизлайк |
| `POST` | `/api/v1/posts/{post_id}/comments` | 201 | Добавить комментарий (см. `comments-prd.md`) |
| `GET` | `/api/v1/posts/{post_id}/comments` | 200 | Комментарии поста (см. `comments-prd.md`) |
| `GET` | `/api/v1/users/{user_id}/likes?type=post` | 200 | Лайкнутые посты пользователя |

## Out of Scope (v1)

- **Черновики** — посты сразу публичные
- **Приватные посты** — видимость в будущем
- **Поиск** — полнотекстовый поиск по постам в следующей итерации
- **Персонализация** — следующая итерация
- **Лента постов конкретного пользователя** — в следующей итерации
- **Лимиты на количество фотографий** — установить до масштабирования

## Risk Assessment

| Риск | Вероятность | Влияние | Митигация |
|------|-------------|---------|-----------|
| Накрутка рейтинга | Средняя | Среднее | Одна реакция на пользователя на уровне БД |
| Посты с ссылками на удалённые рецепты | Высокая | Среднее | Плейсхолдер на клиенте; `recipe_status` в API |
| Спам и злоупотребления | Средняя | Высокое | Аналогично статьям — репорты и админское удаление |
| Отсутствие лимита на количество фотографий | Средняя | Среднее | Установить лимиты до масштабирования |
