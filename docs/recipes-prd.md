**Version**: 1.0
**Date**: 2026-04-07
**Author**: Sarah (Product Owner)

---

## Executive Summary

Рецепты — базовый инструмент Roasti: структурированные инструкции заваривания с пошаговым таймером. Пользователи создают личные рецепты, делятся публичными, клонируют чужие и лайкают понравившиеся. Это основная утилитарная фича платформы, отличающая Roasti от обычных соцсетей.

---

## Problem Statement

**Текущая ситуация:** Рецепты заваривания разбросаны по заметкам и мессенджерам — нет единого специализированного инструмента с поддержкой таймера и структурированных шагов.

**Предлагаемое решение:** Структурированные рецепты с пошаговыми инструкциями, опциональным таймером на каждый шаг и режимом заваривания; поиск и фильтрация по ключевым параметрам; возможность делиться и клонировать публичные рецепты.

**Бизнес-эффект:** Удержание пользователей через утилитарную ценность (brew timer); рост контентной базы за счёт клонирования и публикации рецептов.

---

## User Personas

### Кофейный энтузиаст
- **Цели:** Хранить личную коллекцию рецептов, пробовать чужие, заваривать по шагам с таймером
- **Поведение:** Создаёт рецепты, клонирует понравившиеся, использует brew timer во время заваривания

### Обжарщик / Кофейня
- **Цели:** Публиковать рецепты под конкретные лоты зёрен, рекомендовать способы заварки своей аудитории
- **Поведение:** Создаёт публичные рецепты, привязывает к зёрнам, прикрепляет к постам в ленте

## User Stories & Acceptance Criteria

### Story 1: Создание рецепта

**As a** авторизованный пользователь  
**I want to** создать структурированный рецепт заваривания  
**So that** хранить его в своей коллекции и/или поделиться с сообществом

**Acceptance Criteria:**
- [ ] Рецепт требует: название, метод заварки, сложность
- [ ] Опциональные поля: описание, уровень обжарки, зерно (`bean_id`), приватная заметка (только автору), публичность (по умолчанию — публичный)
- [ ] Рецепт содержит упорядоченный список шагов; список может быть пустым
- [ ] Каждый шаг имеет: заголовок, описание (опционально), длительность в секундах (опционально), изображение (опционально)
- [ ] Любой авторизованный пользователь может создать рецепт

### Story 2: Просмотр и поиск рецептов

**As a** пользователь  
**I want to** искать рецепты по тексту и фильтровать по параметрам  
**So that** находить рецепты под нужный метод заварки, зерно или уровень сложности

**Acceptance Criteria:**
- [ ] `GET /api/v1/recipes` возвращает публичные рецепты + приватные рецепты текущего пользователя
- [ ] Поиск по тексту (`query`): по названию и описанию
- [ ] Фильтрация по: `brew_method`, `difficulty`, `roast_level`, `author_id`
- [ ] Сортировка по дате (новые первые) и по лайкам
- [ ] Результат пагинирован; каждый рецепт содержит `is_liked`

### Story 3: Редактирование и удаление рецепта

**As a** автор рецепта  
**I want to** редактировать или удалить свой рецепт  
**So that** поддерживать актуальность своего контента

**Acceptance Criteria:**
- [ ] Только автор может редактировать и удалять рецепт (403 для остальных)
- [ ] Редактирование возвращает обновлённый объект рецепта
- [ ] Удаление необратимо; возвращает 204

### Story 4: Клонирование рецепта

**As a** пользователь  
**I want to** клонировать понравившийся публичный рецепт  
**So that** адаптировать его под себя, не меняя оригинал

**Acceptance Criteria:**
- [ ] Любой авторизованный пользователь может клонировать чужой публичный рецепт
- [ ] Нельзя клонировать собственный рецепт (403)
- [ ] Клон — независимая копия; изменения в оригинале не влияют на клон
- [ ] Клон хранит ссылку на оригинал (`origin_recipe_id`)

### Story 5: Лайк рецепта

**As a** пользователь  
**I want to** лайкнуть или убрать лайк с рецепта  
**So that** сохранить понравившийся рецепт и выразить отношение к контенту

**Acceptance Criteria:**
- [ ] Toggle-логика: повторный запрос убирает лайк
- [ ] Ответ содержит `liked` и актуальный `likes_count`
- [ ] Лайкнуть может любой авторизованный пользователь, включая автора

### Story 6: Лента лайкнутых рецептов

**As a** пользователь  
**I want to** просмотреть рецепты, которые я лайкнул  
**So that** вернуться к понравившемуся контенту

**Acceptance Criteria:**
- [ ] `GET /api/v1/users/{user_id}/likes?type=recipe` возвращает пагинированный список `LikedRecipe`
- [ ] Каждый элемент содержит `liked_at` и полный объект рецепта

## Functional Requirements

### Поля рецепта

| Поле | Тип | Обязательное | Описание |
|------|-----|--------------|----------|
| `title` | string | да | Название рецепта |
| `brew_method` | enum | да | Метод заварки: v60, aeropress, chemex, french_press, cold_brew, espresso_machine, moka_pot |
| `difficulty` | enum | да | Сложность: easy, medium, hard |
| `description` | string | нет | Описание рецепта |
| `roast_level` | enum | нет | Уровень обжарки |
| `bean_id` | string | нет | Ссылка на зерно из каталога |
| `private_note` | string | нет | Приватная заметка, видна только автору |
| `is_public` | bool | нет | Публичность (по умолчанию true) |
| `steps` | array | нет | Упорядоченный список шагов |

### Поля шага (BrewStep)

| Поле | Тип | Обязательное | Описание |
|------|-----|--------------|----------|
| `title` | string | да | Заголовок шага |
| `description` | string | нет | Описание шага |
| `duration_seconds` | int | нет | Длительность для таймера |
| `image_id` | string | нет | Изображение шага |

### Видимость рецептов

| Тип рецепта | Кто видит |
|-------------|-----------|
| Публичный | Все авторизованные пользователи |
| Приватный | Только автор |

Приватный рецепт недоступен другим пользователям: `GET /api/v1/recipes/{id}` возвращает 404 (не 403) для избежания утечки существования.

### Права доступа

| Действие | Кто может |
|----------|-----------|
| Создать рецепт | Любой авторизованный пользователь |
| Читать публичный рецепт | Любой авторизованный пользователь |
| Читать приватный рецепт | Только автор |
| Редактировать рецепт | Только автор |
| Удалить рецепт | Только автор |
| Клонировать рецепт | Любой авторизованный пользователь (не автор) |
| Лайкнуть рецепт | Любой авторизованный пользователь |

### Brew Timer

Интерактивный пошаговый режим заваривания — полностью клиентская фича, API-изменений не требует:
- Запускает таймер обратного отсчёта на `duration_seconds` текущего шага
- Кнопки: «Старт», «Следующий шаг», «Пауза»
- Keep-screen-awake во время сеанса
- Прогресс-бар по шагам

## API Contract

| Метод | Путь | Статус | Описание |
|-------|------|--------|----------|
| `GET` | `/api/v1/recipes` | 200 | Список рецептов (поиск, фильтрация, пагинация) |
| `POST` | `/api/v1/recipes` | 201 | Создать рецепт |
| `GET` | `/api/v1/recipes/{recipe_id}` | 200 | Получить рецепт |
| `PUT` | `/api/v1/recipes/{recipe_id}` | 200 | Обновить рецепт |
| `DELETE` | `/api/v1/recipes/{recipe_id}` | 204 | Удалить рецепт |
| `POST` | `/api/v1/recipes/{recipe_id}/like` | 200 | Toggle лайк |
| `POST` | `/api/v1/recipes/{recipe_id}/clone` | 201 | Клонировать рецепт |
| `POST` | `/api/v1/recipes/{recipe_id}/comments` | 201 | Добавить комментарий (см. `comments-prd.md`) |
| `GET` | `/api/v1/recipes/{recipe_id}/comments` | 200 | Комментарии рецепта (см. `comments-prd.md`) |
| `GET` | `/api/v1/users/{user_id}/likes?type=recipe` | 200 | Лайкнутые рецепты пользователя |

## Out of Scope (v1)

- **Черновики рецептов** — рецепты сразу публикуются; черновики в следующей итерации
- **Версионирование рецептов** — история изменений
- **Реакции на рецепт** — только лайки в v1
- **Пагинация шагов** — шаги отдаются единым списком
- **Ограничение количества шагов** — лимиты установить до масштабирования
