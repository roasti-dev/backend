---
name: Discuss before creating documents
description: When user asks to write a PRD or similar document, discuss design decisions first — don't immediately create the file
type: feedback
---

Don't jump straight to creating PRD files (or similar documents) when given rough notes. First discuss open design questions with the user, then write once aligned.

**Why:** User gave a rough sketch for posts PRD and I immediately created the file without discussing API design, edge cases, or open questions first.

**How to apply:** When user says "let's write a PRD" with notes, ask about the open design questions first (API shape, edge cases, scope decisions), then create the file after alignment.
